## Q7: Recurring friction points that could be eliminated with new workflows

**1. Session startup lag — 5-7 minutes of reading before being useful.** Every session I read SOUL.md, USER.md, MEMORY.md, daily files, shifts/state.json. This is necessary but repeatable. A session-startup script that pre-fetches and summarizes "what changed since last session" (git diff of memory/, last 3 daily entries, shifts state) would cut this to 30 seconds. The three-shifts system partially solves this by having the planner read context, but the main session still cold-boots.

**2. CTA integration blockers — 4 tracks stuck waiting on external parties.** Andy (web version URL), Eric (ClawBox affiliate URL), Lovable (access to edit CTAs), David (mobile app approval). Each requires a different communication channel and follow-up cadence. A blocker-tracking system in MEMORY.md with explicit owners, last-check dates, and next-action dates would replace the current "buried in daily files" pattern. Night shift could auto-check: "Has it been 3+ days since we pinged about X? Add to handoff."

**3. Cron health visibility — failures are silent until manually checked.** The venice-key-monitor and venice-402-watchdog run on schedules, but when key3 ran dry, David noticed before I did. I had log files but hadn't built proactive surfacing. A nightly cron-health summary appended to handoff.md (last run timestamps, error counts, balance warnings) would make the night shift output immediately actionable.

**4. Multi-repo sync is manual and error-prone.** Push to origin → push to 28 flavors → PR to org → David merges. The push-to-all.sh script exists, but it's invoked manually. If I forget one step, releases are partial. A release-checklist skill that: (1) verifies all remotes match origin commit, (2) opens PR with correct version, (3) confirms merge — would reduce the "did I push to all 28?" anxiety.

**5. Git working directory drift — 37 modified files, nothing staged.** The workspace has accumulated modifications (SOUL.md changes, gateway-guardian.sh, subrepos) but no clear commit strategy. Night shift could auto-commit "WIP: night shift context updates" for non-sensitive changes, keeping the working directory clean. Currently I'm leaving a mess for the next session.
