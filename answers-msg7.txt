## Q19: Workflows David still does manually that could be automated

**1. Flavor repo sync after releases.** After each EverClaw release, David manually runs push-to-all.sh, verifies each of the 28 flavor repos, opens a PR to the org repo, and merges. This is a scripted git workflow that could be fully automated: trigger on version tag, cascade to all flavors, open PR with changelog, and send a Signal notification when ready for merge. Time saved: ~30 minutes per release, ~2-3 hours/month.

**2. Finance/balance tracking.** David asks for balance updates. We have a finance tracker cron, but it runs on demand. A scheduled daily snapshot (morning and evening) with a concise Signal summary would replace the "where are we on X?" requests. The daily snapshot could include: MOR balance, USDC Balances, net worth calculation, trajectory toward $1B goal.

**3. Morpheus session health checks.** When the proxy-router stalls, David notices inference failures before I do. A 15-minute cron that: pings localhost:8082/health, tests a simple inference, verifies session is active, and notifies via Signal if degraded would catch issues before David hits them.

**4. School calendar reminders.** We have the RRISD calendar file, but David manually tracks "is tomorrow a school day?" or "when's the next early release?" A cron that runs Sunday nights and sends: "This week: normal Mon-Thu, early release Fri (teacher workday), no school next Monday (Presidents Day)" would replace manual calendar checking.

**5. Key rotation monitoring.** We have 6 Venice keys with different balances. David has to ask "which key is low?" A rotating key status in the daily Signal summary would surface this proactively: "key1: 45 DIEM, key2: 38 DIEM, key3: 0.91 USD — needs refill soon."

**6. Meeting preparation notes.** When David has a meeting (e.g., Friday's active staking call), he likely reviews context beforehand. I could auto-generate prep notes 2 hours before each known meeting: participants, context from memory, recent relevant decisions, and suggested talking points. This would replace manual review.

**1. Memory system was just overhauled, but the structure is still flat.** We have 100+ files across marketing/, projects/, daily/, reference/. The new hybrid search helps find things, but organization is still scattered. But I don't reference them when answering questions about flavor strategy. The CTA integration blockers are about launching these plans — but the plans sit dormant. A rebuild would: (a) extract the actionable parts (CTAs, channel targets, timelines) into a tracking system, (b) link each blocker to the specific marketing step it blocks, (c) make the files part of the weekly heartbeat check.

**3. **5.
