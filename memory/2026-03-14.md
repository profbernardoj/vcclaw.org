## Ollama Routing Bug (Mar 13, major finding)

**Bug:** SpaceAI agent failed with "All models failed" — the ollama/qwen3.5:9b fallback returned "Morpheus session unavailable" error (impossible from localhost:11434).

**Root cause found in OpenClaw source code (~166K lines of minified JS):**
- `scripts/everclaw-wallet.mjs` and config templates had `api: "openai-completions"` for ollama provider
- OpenClaw's routing at line ~107629 checks `if (params.model.api === "ollama")` to set up correct baseUrl
- With `api: "openai-completions"`, this condition is FALSE → falls through to `streamSimple`
- `streamSimple` reuses the HTTP client from the previous provider (morpheus proxy on port 8083)
- Both morpheus and ollama produced identical error hash `sha256:a04dbc7e62e4` — confirmed same endpoint hit

**Fixes applied:**
1. **Our OpenClaw config:** Patched `ollama.api` from `"openai-completions"` to `"ollama"` via gateway config.patch
2. **All 5 secondary agents:** Updated to `venice/claude-opus-4-6` primary, `mor-gateway/glm-5` + `ollama/qwen3.5:9b` fallbacks. Removed `morpheus/kimi-k2-thinking` (no bids on P2P network).
3. **EverClaw templates:** Fixed `api` field in `openclaw-config-mac.json`, `openclaw-config-linux.json`, `scripts/setup-ollama.sh` — commit `cbb9c4f`, all 30 repos synced.
4. **Existing user migration:** Added auto-migration to 3 paths:
   - `setup-ollama.sh` — `migrate_ollama_api()` in configure step
   - `install-with-deps.sh` — post-merge migration for curl|bash upgrades
   - `setup.mjs` — runs on every `--apply`
   Commit `73a5cf8`, all 30 repos synced.

**GitHub issue:** Filed openclaw/openclaw#45369, then discovered fix already exists in upstream v2026.3.13 (`isOllamaCompatProvider()` function). Issue closed as already fixed. Our v2026.3.12 needs the workaround (api: "ollama") which we've applied.
