## Lessons Learned

1. **MOR is STAKED, not spent** — `directPayment:false` confirmed in router logs. All staked MOR returns after session close. Only ETH gas is consumed. MOR does NOT "run out."
2. **Session stake is pro-rated by duration** — 4,434 MOR/7 days = 633 MOR/day (all returned after close)
3. **Need enough for one session at a time** — MOR doesn't "run out"; you need enough unlocked MOR to cover the next session's stake
4. **MOR must be in the EOA wallet, not Safe** — Safe requires multi-sig, router only uses EOA
5. **glm-5 not on Morpheus P2P** — No providers offer it; use Gateway
6. **Balance check should block sessions proactively** — Prevents cryptic "transfer exceeds balance" errors
7. **SESSION_CLOSER auto-closes** — Router has built-in component that auto-closes expired sessions and triggers MOR return
8. **Safe smart accounts (1-of-2)** — Single owner can execute transfers via scripts/safe-transfer.mjs
