### Key Decisions Locked
- SDK: `@xmtp/agent-sdk@^2.3.0` (not raw node-sdk)
- Always-on daemon via launchd/systemd (separate process from OpenClaw)
- Filesystem bridge for IPC (outbox/inbox dirs, not HTTP API)
- Consent → Guard → Router middleware chain order
- `Agent.createFromEnv()` with env vars + CWD save/restore
- Two-tier identity: 28 flavor canonical wallets + per-user wallets
- Phase 1 fees = free (payer.mjs is a stub)
- Local `.secrets.json` (chmod 600) for v1 key storage
- `AGENT_CHAT_XMTP_DIR` env var for test isolation + Docker path override

### Critical Bugs Caught During Review
- Bagman is a SKILL (wraps `op` CLI), not importable JS module
- `Agent.createFromEnv()` takes NO args (reads env vars)
- `fs.watch()` from `node:fs/promises` returns AsyncIterator, not callback
- `agentInstance.sendMessage()` doesn't exist — need conversation lookup + `sendText()`
- `createCommsGuardMiddleware()` must be singleton (not per-message)
- `comms-guard storage.init(address)` required before first use
- Health.mjs had dangling Promise for inboxId (serialized as `{}`)
- **Path traversal via correlationId** — `path.join()` resolves `../` allowing writes outside inbox dir
